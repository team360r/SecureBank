<QuizQuestion
  question="Why should the security-scan job be configured as a required status check with branch protection?"
  options={[
    "It makes the CI pipeline run faster by prioritising security checks",
    "It prevents any code from merging to main unless all security checks pass, removing reliance on human discipline",
    "It automatically fixes security issues before the code is merged",
    "It is required by GitHub's terms of service for repositories containing sensitive code"
  ]}
  correctIndex={1}
  explanation="Branch protection with required status checks means that a PR cannot be merged unless the security-scan job passes. This removes the human factor — no matter how rushed a developer is, they cannot bypass static analysis, secret scanning, or dependency auditing. The pipeline becomes the security perimeter."

<QuizQuestion
  question="During a manual penetration test of a Flutter banking app, which check verifies that certificate pinning is working?"
  options={[
    "Decompiling the APK and searching for certificate hash strings in the binary",
    "Running the app through a proxy like Burp Suite and confirming that HTTPS requests are rejected when intercepted",
    "Checking that the app uses the https:// scheme in all API URL constants",
    "Verifying that the app displays a padlock icon in the UI during network requests"
  ]}
  correctIndex={1}
  explanation="The definitive test for certificate pinning is running the app through an intercepting proxy (Burp Suite, mitmproxy, Charles). If pinning is working correctly, the app will refuse to complete TLS handshakes with the proxy's certificate, and all API requests will fail. Simply using HTTPS or checking URL strings does not verify that pinning is enforced."
